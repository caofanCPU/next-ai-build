# 1. CSS

负责：

# “主体空间”

包括：

* 布局
* 曲率
* 毛玻璃
* 阴影

# 2. SVG

SVG 真正负责：

# “结构语言”

即：

* 不规则轮廓
* HUD线
* 参数化边框
* 引导线
* 科技轮廓
* 曲率路径
* 工业切角
* 流体边缘

SVG：

不是主角。

而是：

# 精修刀。

---

# 四、为什么 SVG 是辅助

因为：

# SVG 不适合做主体布局。

原因：

## 1. 响应式困难

复杂 path：

一缩放就变形。

---

## 2. 可维护性差

后期：

没人敢改。

---

## 3. 文档流隔离

SVG：

天然脱离 DOM 流。

---

## 4. 动态内容困难

内容变化时：

SVG 结构容易崩。

---

# 五、真正现代的做法

真正高级产品：

通常：

# HTML/CSS 是壳体

SVG：

只负责：

* 边缘
* 轮廓
* 特征件
* 局部结构

例如：

```tsx id="sj8a7d"
<div className="panel">
  <svg className="accent-line" />
  <svg className="corner-detail" />
  <Content />
</div>
```

而不是：

```tsx id="q8bjlv"
<svg>
  整个页面
</svg>
```
